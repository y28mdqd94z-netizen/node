# Agent OS (Repo)

This repo is a portable Agent OS you can drop into any codebase to standardize how agents plan, design, build, and document work. It ships with an Orchestrator-driven workflow, task logging, and two knowledgebases (UX/UI + engineering) to keep decisions consistent and audit‑ready.

**Capabilities**
- Orchestrated workflow with phase gates and routing notes.
- Task logs and structured outputs in `memory/`.
- UX/UI knowledgebase with laws, principles, patterns, components, and evaluation checklists.
- Engineering knowledgebase with ranked, overrideable guidance for architecture, frontend, backend, security, and ops.
- Design‑system guidance and tokens under `.agent-os/lib/ux-ui-knowledgebase/` and `.agent-os/tokens/`.

**Install**
1. To use this repo directly:
   - Clone it and run the agent commands from the root.
2. To add Agent OS to an existing repo:
   - Copy `.agent-os/` and `memory/` into your repo root.
   - Copy `AGENTS.md` to enforce the rules repo‑wide.
   - Run `./.agent-os/agent bootstrap-project` to initialize memory files and scan the repo.

**Quick Start**
```bash
./.agent-os/agent help
./.agent-os/agent bootstrap-project
./.agent-os/agent scan-repo
```

**Required Orchestration**
- The Orchestrator is mandatory for every task.
- Task logs and outputs must include routing notes and phase gate status.

**Knowledgebases**
- UX/UI knowledgebase: `.agent-os/lib/ux-ui-knowledgebase`
- UX/UI index: `memory/ux_kb_index.json` (generated by `kb-update`)
- UX/UI manifest: `.agent-os/lib/ux-ui-knowledgebase/manifest.json` (updated by `kb-update`)
- Engineering knowledgebase: `.agent-os/lib/engineering-knowledgebase`
- Engineering manifest: `.agent-os/lib/engineering-knowledgebase/manifest.json`

After editing the UX/UI knowledgebase, run:
```bash
./.agent-os/agent kb-update
```

**Codex (VS Code) Integration**
- Root `AGENTS.md` enforces the Agent OS rules for this repo.
- Keep `.agent-os/system.md` as the primary operating spec.
- For UX/UI work, use the knowledgebase and cite entries by file + id/title.

**Setup (VS Code)**
- Install the Codex IDE extension from the VS Code Marketplace.
- Codex appears in the right sidebar by default; restart VS Code if it does not appear.
- Sign in with a ChatGPT account or an API key (ChatGPT plans include usage credits).
- Windows support is experimental; best experience is via WSL.
- Choose an approval mode in the extension (Chat, Agent, Agent Full Access).
- Use editor context (open files, selections, `@file`) for better results.
- Optional: set Codex keyboard shortcuts via the Codex chat settings.
