## 3. DESIGN DECISION
- Think: prioritize clarity, brevity, and task-first phrasing aligned with Calm Systems UI tone; avoid noisy or generic copy.
- UX design: two primary microcopy tasks
  - Task 1: Write labels/CTAs with verb-first actions and explicit outcomes.
  - Task 2: Write state messaging (empty/error/success) that explains cause and next steps.
- Apply Hick/Miller/Jakob/Tesler
  - Hick: reduce choice ambiguity by using specific labels instead of generic CTAs.
  - Miller: keep helper text and state copy short and chunked.
  - Jakob: use familiar, conventional wording for common actions.
  - Tesler: avoid pushing complexity into the copy; simplify flows when possible.
- States, transitions, feedback, empty/error/loading
  - States copy should follow existing feedback tone guidance and stay short, actionable, and blame-free.
