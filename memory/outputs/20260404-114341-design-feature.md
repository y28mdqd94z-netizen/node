## 3. DESIGN DECISION
Section 3: Current-State Audit (Agent OS design layer)
- Strengths:
  - Strong UX/UI knowledgebase with laws, principles, patterns, components, and evaluation checklists.
  - Clear Calm Systems UI signature and content-writing guidance.
  - Token system exists in `.agent-os/tokens/` and `memory/design_system.json`.
- Weaknesses / gaps:
  - No persistent design-context file (missing teach-impeccable analog).
  - Retrieval is knowledgebase-centric; design guidance is not modular for quick, low-token loading.
  - No command-style design workflows (critique, polish, distill) as dedicated skills.
  - No design-layer manifest for ranked, overrideable retrieval.
- Duplication / monolith risk:
  - Some guidance is embedded in large YAMLs; design use-cases require curated slices.

Section 4: Gap Analysis vs Impeccable and benchmarks
- Impeccable does better:
  - Modular references (7 files), explicit anti-patterns, command skills, and context-capture workflow.
  - Single-source-of-truth philosophy with provider transforms (not adopted here by choice).
- Agent OS does better:
  - Richer UX law/pattern taxonomy and a stronger evaluation checklist library.
  - Calm Systems UI signature and content-writing rules are more systemized than most skills.
- Borrow / adapt:
  - Context-capture protocol (teach-impeccable pattern) -> `memory/design_context.yaml`.
  - Modular references -> `.agent-os/design/references/` thin, ranked files.
  - Command workflows -> `.agent-os/design/workflows/`.
- Avoid:
  - Full build pipeline (deferred by choice to keep portability simple).

Section 5: Proposed Design Architecture (Additive Overlay)
- `.agent-os/design/core/` -> philosophy, principles, context model
- `.agent-os/design/skills/` -> foundational and specialist skills (Agent Skills format)
- `.agent-os/design/workflows/` -> command-style skills (critique/audit/polish/normalize/distill/clarify/typeset/arrange)
- `.agent-os/design/references/` -> modular, ranked references
- `.agent-os/design/anti-patterns/` -> explicit anti-pattern files
- `.agent-os/design/evals/` -> operational rubrics/checklists
- `.agent-os/design/templates/` -> design context + critique/audit templates
- `.agent-os/design/manifest.json` -> ranked index for retrieval

Section 6: Foundational Design Skill Redesign
- New foundational skill: `design-core` (equivalent to `frontend-design`).
- Contents:
  - Context Gathering Protocol (check and update `memory/design_context.yaml`).
  - Aesthetic direction commitment (avoid generic defaults).
  - Guardrails: accessibility baseline, hierarchy, component reuse.
  - Reference navigation and anti-pattern checks.
- Detailed guidance moved to references:
  - typography, color/contrast, spacing/layout, motion, interaction, responsive, UX writing.

Section 7: Design Context System (teach-impeccable inspired)
- New persistent file: `memory/design_context.yaml` (template in `.agent-os/design/templates/`).
- Captures: purpose, users, brand personality, principles, interaction prefs, platform constraints, accessibility, visual direction, motion, content tone, differentiation goals.
- Design skills must request and store missing fields before proceeding.
