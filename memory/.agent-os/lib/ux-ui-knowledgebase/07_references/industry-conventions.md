# Industry Conventions

## Navigation
- Top navigation for 3-6 primary sections on desktop.
- Sidebar navigation for deep hierarchies and admin tools.
- Bottom navigation for 3-5 primary destinations on mobile.

## Forms
- Single-column forms improve scanability and completion.
- Inline validation reduces errors when used after input.
- Provide clear primary and secondary actions.

## Tables and Data
- Keep tables to 6 or fewer columns when possible.
- Provide sorting on common columns and filtering for large datasets.
- Use sticky headers for long tables.

## Feedback
- Use toasts for non-blocking confirmations.
- Use inline alerts for contextual errors.
- Use banners for system-wide messages.

## Accessibility
- Keyboard navigation is required for all interactions.
- Contrast must meet WCAG AA.
- Use text labels in addition to color or icon-only indicators.

## Content
- Use a clear H1 and consistent heading hierarchy.
- Keep line length between 60-80 characters for readability.
- Use concise, action-oriented button labels.
