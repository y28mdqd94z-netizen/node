# Interaction Model (Short-Form)

Rules:
- Define primary + secondary actions per view.
- All actions must show immediate feedback.
- Errors must explain cause and next step.
- Avoid multi-step flows unless necessary; reduce to 1-2 actions per task.

Checks:
- Loading, empty, and error states defined for each primary view.
- Destructive actions require confirmation.
