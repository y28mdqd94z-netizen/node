# Portable Agent OS

This folder is self-contained and safe to drop into any repo. It does not modify existing source files, install dependencies, or access the network.

## Usage

```bash
./.agent-os/agent help
./.agent-os/agent bootstrap-project
./.agent-os/agent scan-repo
```

Commands may be prefixed with `/` (e.g. `/.agent-os/agent /scan-repo`).

## Agents and Skills

- Core agents: `.agent-os/agents.json`
- Orchestrator is mandatory for routing and phase gating
- Specialist micro-skills: `.agent-os/skills.json`
- UX/UI knowledgebase: `.agent-os/lib/ux-ui-knowledgebase` (laws, principles, patterns, components, evaluation checklists)
- Operating spec: `.agent-os/system.md`
- Codex project instructions: `AGENTS.md` (applies repo-wide)

## Orchestration (Required)

- The Orchestrator must be invoked at the start of every task
- Orchestrator decides routing to core agents and enforces phase order
- Outputs and task logs must include routing notes and phase gate status
- If Orchestrator is not invoked, stop and re-run the task

## Outputs

Feature commands create a structured output template in `memory/outputs/` that follows the required format:

1. SUMMARY  
2. MARKET ANALYSIS  
3. DESIGN DECISION  
4. ARCHITECTURE  
5. IMPLEMENTATION  
6. SECURITY REVIEW  
7. NEXT ACTIONS  

## Risk Gating

- `--risk-level low|medium|high` (default: `low`)
- `--confirm-high-risk` required for high risk
- `--title "Your task title"` to name the task/output

## Commands

- `bootstrap-project`  Create memory files and scan repo
- `scan-repo`          Update `memory/repo_map.json` only
- `sync-docs`          Ensure memory files exist (no overwrite)
- `kb-update`          Re-index UX/UI knowledgebase + update manifest
- `design-feature`     Log a design task stub
- `architect-feature`  Log an architecture task + decision stub
- `build-ui`           Log a UI build task stub
- `implement-feature`  Log an implementation task + decision + risk stub
- `refine-ui`          Log a UI refinement task stub
- `ux-audit`           Log a UX audit task + risk stub
- `security-review`    Log a security review task + risk stub
- `compress-context`   Log a context compression task stub
- `help`               Show help
- `version`            Show version

## Knowledgebase

- Location: `.agent-os/lib/ux-ui-knowledgebase`
- Index: `memory/ux_kb_index.json` (generated by `kb-update`)
- Manifest: `.agent-os/lib/ux-ui-knowledgebase/manifest.json` (updated by `kb-update`)
- Read `.agent-os/system.md` before work
- Cite KB entries by file + id/title in UX decisions
- Use `02_design-evaluation/` checklists for audits
- Use `06_agent-application/` rules for pattern/component selection
- After KB edits, run `./.agent-os/agent kb-update`

## Codex (VS Code) Integration

- Root `AGENTS.md` enforces the Agent OS rules for the repo
- Keep `.agent-os/system.md` as the primary operating spec
- For UX/UI work, always use the knowledgebase and cite entries
- Install the Codex IDE extension from the VS Code Marketplace
- After install, Codex appears in the editor sidebar (right side by default in VS Code). Restart VS Code if it does not appear
- Sign in with a ChatGPT account or an API key (ChatGPT plans include usage credits)
- Windows support is experimental; best experience is via WSL
- Choose an approval mode in the extension (Chat, Agent, Agent Full Access)
- Use editor context (open files, selections, @file) to improve prompt quality
- Optional: set Codex keyboard shortcuts via the Codex chat settings

## Portability

To remove the system, delete `.agent-os/` and `memory/`. No other files are touched.
