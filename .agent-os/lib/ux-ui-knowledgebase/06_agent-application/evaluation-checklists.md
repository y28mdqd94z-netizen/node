# Evaluation Checklists

Use these checklists to critique implementations.

## Design quality
- Primary action is visible without scrolling.
- Hierarchy is clear within 3 seconds.
- Spacing follows the scale with consistent rhythm.

## Usability
- Critical tasks complete in <= 2 steps.
- Errors include recovery guidance.
- Empty states provide a next action.

## Accessibility
- Keyboard navigation reaches all actions.
- Focus states are visible and consistent.
- Contrast meets baseline requirements.

## Responsiveness
- Layout works at xs, md, and xl breakpoints.
- Navigation collapses without hiding primary tasks.
