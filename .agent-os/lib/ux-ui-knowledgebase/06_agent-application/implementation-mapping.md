# Implementation Mapping

Use this guide to map primitives into code.

## Token mapping
- Convert base and semantic tokens into CSS variables.
- Use component tokens directly in component styles.

## Component mapping
- One component module per primitive.
- Variants map to component props and styles.

## Pattern mapping
- Patterns map to feature-level composition modules.
- Pattern files declare required components and states.

## Shell mapping
- Shells map to layout containers and navigation scaffolds.
- Shell selection drives default density and breakpoints.
