# Shell Selection Guide

Use shells to determine layout, navigation, and density defaults before selecting patterns.

## Steps
1. Identify product context and navigation depth.
2. Choose the shell that matches primary workflows.
3. Verify compatible patterns in `08_primitives/04_shells`.

## Quick picks
- `portal-shell`: multi-section SaaS and admin portals.
- `dashboard-shell`: KPI and monitoring dashboards.
- `list-management-shell`: CRUD and record management.
- `builder-shell`: editors and builders.
- `focus-shell`: onboarding and single-task flows.
- `mobile-sheet-first-shell`: mobile-first utilities.
