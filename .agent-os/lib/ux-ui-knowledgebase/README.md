# UX/UI Design Knowledgebase

This **knowledgebase** is organized as a modular collection of YAML and Markdown files, optimized for machine retrieval and extensibility. It captures core UX laws, principles, patterns, design system rules, and agent guidance with citations to authoritative sources. Below is the proposed folder structure and sample content for key files.

## Structure

This repository was generated from `design system.md` and follows the proposed folder structure.

## Prioritization of Deliverables

Given the breadth, a **must-have core** would include:

- **Taxonomy** linking categories (laws → principles → patterns → components).
- **Key Law entries** (Hick, Fitts, Jakob, Miller, Tesler, Peak–End, Serial, Von Restorff, Zeigarnik, Doherty).
- **Core Patterns** (navigation, form, modal, onboarding, etc.) with YAML schemas.
- **Core Components** (Button, Input, Select, Dialog, etc.) with usage guidance.
- **Design system foundations** (spacing, colour, typography, tokens).
- **Agent logic** (rules mapping principles to decisions).
- **Source map** for all used guidance.

**Nice-to-have** (for later) includes exhaustive pattern files (e.g. calendars, charts, mobile-specific), additional cognitive psychology deep dives (habit loop, decision fatigue), and more heuristics details.
